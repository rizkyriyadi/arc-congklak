# Arc Entertainment — Product Roadmap

**Owner:** CEO · **Last updated:** 2026-06-24 · **Goal:** Building Indonesian-based indie games

## Strategy in one line

Ship a small, polished, browser-playable game rooted in Indonesian culture — fast — then iterate based on real playtests. Prove we can make something fun before we make something big.

## Why web/HTML5 first

- **Distribution:** a URL is the whole install story. No app stores, no gatekeepers.
- **Verifiability:** QA and the CEO can play it in a browser. Tight feedback loop.
- **Cost & speed:** TypeScript + Canvas (or Phaser if it clearly helps) gets us to "playable" in days, not months.
- **Reversible:** if the first title misses, we've spent little and learned a lot.

## Phase 0 — Foundation (now)

Stand up the repo and a "hello, it runs in a browser" loop so every later task has somewhere to land.

- **P0.1** Scaffold the game project: TypeScript + HTML5 Canvas (or Phaser), build/dev tooling, a blank canvas that renders and runs locally and in a browser. *(→ ARC child issue, FoundingEngineer)*
- **P0.2** Decide the first-title concept: a tight, single-mechanic arcade/puzzle game with an Indonesian cultural theme. One-page concept doc. *(CEO + FoundingEngineer; CEO owns the creative call)*

## Phase 1 — Playable prototype (first title)

A vertical slice that is actually fun to play for 60 seconds.

- **P1.1** Core game loop: the one mechanic, playable start-to-fail/score with keyboard or touch input.
- **P1.2** Indonesian theming pass: visual identity, naming, and setting drawn from Indonesian culture/folklore (CEO signs off on cultural direction).
- **P1.3** Score / win-lose state + a basic title and restart screen.
- **P1.4** Playtest build deployed to a shareable URL; CEO playtests and gives go/no-go.

## Phase 2 — Polish & feel

Turn "it works" into "it feels good."

- Juice: sound, animation, feedback, difficulty curve.
- Art direction pass (UX/Game Designer once hired).
- Performance: smooth on a mid-range laptop and mobile browser.

## Phase 3 — Launch & learn

- Public prototype release on an itch.io-style page or our own URL.
- Lightweight analytics + a way to collect player feedback.
- Marketing/DevRel hire activates here (not before).

## Operating cadence

- Roadmap is a living doc. Each phase's work becomes concrete ARC child issues assigned to the right owner.
- The CEO owns sequencing and creative/cultural calls; the Founding Engineer owns technical execution and flags hiring needs early.
- We do not advance a phase until the prior phase's go/no-go is met.
